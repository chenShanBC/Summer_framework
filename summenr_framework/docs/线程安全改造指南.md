# IoC 容器线程安全改造指南

## 一、问题分析

### 当前存在的线程安全问题

1. **HashMap 不是线程安全的**
   - `beanDefinitionMap` 和 `singletonObjects` 使用 HashMap
   - 多线程并发读写可能导致数据不一致或死循环

2. **getBean() 方法的竞态条件**
   - 多个线程同时调用 `getBean()` 可能重复创建 Bean
   - 即使使用 ConcurrentHashMap，也需要保证创建过程的原子性

3. **creatingBeanNames 的线程安全**
   - HashSet 不是线程安全的
   - 循环依赖检测可能失效

---

## 二、改造方案

### 方案 1：最小改动（推荐用于学习）

**适用场景：** 所有 Bean 在容器启动时创建完成（饿汉式）

**改动点：**
- 将 HashMap 改为 ConcurrentHashMap
- 将 HashSet 改为 ConcurrentHashMap.newKeySet()

**优点：** 改动最小，性能好
**缺点：** 不支持懒加载

---

### 方案 2：完整线程安全（推荐用于生产）

**适用场景：** 支持懒加载，Bean 可以在运行时动态创建

**改动点：**
- 使用 ConcurrentHashMap
- getBean() 方法使用双重检查锁定（DCL）
- 使用 synchronized 保证 Bean 创建的原子性

**优点：** 完全线程安全，支持懒加载
**缺点：** 代码稍复杂

## 三、具体实现步骤

### 步骤 1：修改 ClassPathXmlApplicationContext.java

**位置：** `src/main/java/com/wubai/summer/core/ClassPathXmlApplicationContext.java`

#### 1.1 修改成员变量声明（第 21-23 行）

**原代码：**
```java
private final Map<String, BeanDefinition> beanDefinitionMap = new HashMap<>();
private final Map<String, Object> singletonObjects = new HashMap<>();
private final Set<String> creatingBeanNames = new HashSet<>();
```

**改为：**
```java
private final Map<String, BeanDefinition> beanDefinitionMap = new ConcurrentHashMap<>();
private final Map<String, Object> singletonObjects = new ConcurrentHashMap<>();
private final Set<String> creatingBeanNames = ConcurrentHashMap.newKeySet();
```

**原因：**
- ConcurrentHashMap 是线程安全的 Map 实现
- ConcurrentHashMap.newKeySet() 返回线程安全的 Set

---

#### 1.2 修改 getBean() 方法（第 55-92 行）

**原代码：**
```java
public Object getBean(String beanName) {
    // 1. 单例池中有则直接返回
    if (singletonObjects.containsKey(beanName)) {
        return singletonObjects.get(beanName);
    }

    // 2. 获取BeanDefinition
    BeanDefinition beanDef = beanDefinitionMap.get(beanName);
    if (beanDef == null) {
        throw new RuntimeException("Bean不存在：" + beanName);
    }

    // 3. 循环依赖检测
    if (!creatingBeanNames.add(beanName)) {
        throw new RuntimeException("检测到循环依赖：" + beanName);
    }

    // ... 后续创建逻辑
}
```

**改为（双重检查锁定）：**
```java
public Object getBean(String beanName) {
    // 第一次检查：快速路径，无锁
    Object bean = singletonObjects.get(beanName);
    if (bean != null) {
        return bean;
    }

    // 获取BeanDefinition
    BeanDefinition beanDef = beanDefinitionMap.get(beanName);
    if (beanDef == null) {
        throw new RuntimeException("Bean不存在：" + beanName);
    }

    // 同步块：保证只有一个线程创建Bean
    synchronized (beanDef) {
        // 第二次检查：防止重复创建
        bean = singletonObjects.get(beanName);
        if (bean != null) {
            return bean;
        }

        // 循环依赖检测
        if (!creatingBeanNames.add(beanName)) {
            throw new RuntimeException("检测到循环依赖：" + beanName);
        }

        System.out.println("🔨 实例化Bean：" + beanName);

        // 实例化Bean
        Object instance;
        if (beanDef.getFactoryMethod() == null) {
            instance = instantiateByConstructorXml(beanDef);
        } else {
            instance = instantiateByFactoryMethod(beanDef);
        }

        // 属性注入
        injectPropertiesXml(instance, beanDef);

        // 放入单例池
        singletonObjects.put(beanName, instance);
        creatingBeanNames.remove(beanName);

        return instance;
    }
}
```

**关键点：**
- **第一次检查**：无锁，快速返回已存在的 Bean
- **synchronized (beanDef)**：以 BeanDefinition 为锁，不同 Bean 可以并发创建
- **第二次检查**：防止多个线程同时进入同步块后重复创建

## 四、验证方法

### 4.1 编写多线程测试

创建测试文件：`src/test/java/com/wubai/summer/test/ThreadSafetyTest.java`

```java
package com.wubai.summer.test;

import com.wubai.summer.core.ClassPathXmlApplicationContext;
import com.wubai.summer.test.Services.UserService;

import java.util.concurrent.CountDownLatch;
import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;

public class ThreadSafetyTest {
    public static void main(String[] args) throws InterruptedException {
        ClassPathXmlApplicationContext context =
            new ClassPathXmlApplicationContext("beans.xml");

        int threadCount = 100;
        CountDownLatch latch = new CountDownLatch(threadCount);
        ExecutorService executor = Executors.newFixedThreadPool(10);

        // 100个线程同时获取Bean
        for (int i = 0; i < threadCount; i++) {
            executor.submit(() -> {
                try {
                    UserService service = (UserService) context.getBean("userService");
                    System.out.println(Thread.currentThread().getName() + ": " + service);
                } finally {
                    latch.countDown();
                }
            });
        }

        latch.await();
        executor.shutdown();
        System.out.println("✅ 所有线程执行完成");
    }
}
```

### 4.2 预期结果

- 所有线程获取到的是同一个 UserService 实例（地址相同）
- 不会抛出异常
- 不会出现死锁或卡死

## 五、性能优化建议

### 5.1 锁粒度优化

**当前方案：** `synchronized (beanDef)` - 按 Bean 加锁

**优点：** 不同 Bean 可以并发创建
**缺点：** 同一个 Bean 的多次获取会串行

**进一步优化：** 使用 ReentrantLock 或 Semaphore

### 5.2 饿汉式 vs 懒加载

**饿汉式（当前实现）：**
- 容器启动时创建所有 Bean
- 启动慢，运行快
- 线程安全问题较少

**懒加载：**
- 第一次 getBean() 时才创建
- 启动快，首次访问慢
- 必须处理好线程安全

## 六、常见问题

### Q1: 为什么不直接用 synchronized 方法？

```java
public synchronized Object getBean(String beanName) { ... }
```

**问题：** 所有 Bean 的获取都会串行，性能差

**解决：** 使用双重检查锁定 + 细粒度锁（按 BeanDefinition 加锁）

### Q2: ConcurrentHashMap 就够了吗？

**不够！** ConcurrentHashMap 只保证单个操作的原子性，但 Bean 创建是多步操作：
1. 检查是否存在
2. 创建 Bean
3. 放入容器

这三步必须作为一个整体加锁。

### Q3: 为什么用 synchronized (beanDef) 而不是 synchronized (this)？

**synchronized (this)：** 锁住整个容器，所有 Bean 串行创建
**synchronized (beanDef)：** 锁住单个 Bean，不同 Bean 可以并发创建

## 七、扩展阅读

### Spring 的三级缓存

Spring 使用三级缓存解决循环依赖：
1. **singletonObjects**：完整的 Bean
2. **earlySingletonObjects**：半成品 Bean（已实例化，未注入）
3. **singletonFactories**：Bean 工厂

你的实现目前只有一级缓存，不支持循环依赖。

### 面试要点

- **为什么用 ConcurrentHashMap？** 线程安全 + 高并发性能
- **什么是双重检查锁定？** 减少锁竞争，提高性能
- **单例 Bean 线程安全吗？** 取决于 Bean 是否有状态
- **如何测试线程安全？** 多线程并发测试 + 压力测试
